01 LED Blink
output
▾

What students learn

GPIO digital output, setup() vs loop(), delay() timing — the foundation of all hardware projects.

Wiring connections

Component PinESP32 PinNotes
LED (+) AnodeGPIO 2Through 220Ω resistor
LED (–) CathodeGNDShort leg of LED

Arduino code

Arduino / C++
// Project 1: LED Blink
// GPIO 2 is also the built-in LED on most ESP32 boards

const int LED_PIN = 2;

void setup() {
  pinMode(LED_PIN, OUTPUT);   // set pin as output
}

void loop() {
  digitalWrite(LED_PIN, HIGH);  // LED ON
  delay(1000);                   // wait 1 second
  digitalWrite(LED_PIN, LOW);   // LED OFF
  delay(1000);                   // wait 1 second
}
Try changing 1000 to 100 — the LED blinks 10x faster. Ask students: what happens at delay(10)?

02 Push Button + LED
output logic
▾

What students learn

Digital input reading, pull-up resistors, if-else logic — responding to real-world input.

Wiring connections

Component PinESP32 PinNotes
LED Anode (+)GPIO 2Through 220Ω resistor
LED Cathode (–)GND
Button Pin 1GPIO 4Any leg on one side
Button Pin 2GNDOther leg

Arduino code

Arduino / C++
// Project 2: Push Button controls LED
// INPUT_PULLUP uses ESP32's internal resistor — no external resistor needed

const int LED_PIN    = 2;
const int BUTTON_PIN = 4;

void setup() {
  pinMode(LED_PIN,    OUTPUT);
  pinMode(BUTTON_PIN, INPUT_PULLUP); // internal pull-up: HIGH when not pressed
}

void loop() {
  int btnState = digitalRead(BUTTON_PIN);

  if (btnState == LOW) {    // LOW = button pressed (pulled to GND)
    digitalWrite(LED_PIN, HIGH);
  } else {
    digitalWrite(LED_PIN, LOW);
  }
}
INPUT_PULLUP inverts logic: LOW means pressed. Ask students why this is and what a pull-down resistor does differently.

Wiring connections

Component PinESP32 PinNotes

Potentiometer Middle (Wiper)GPIO 34Analog input pin
Potentiometer RightGNDGround pin

Arduino code

Arduino / C++
// Project 3: Potentiometer value on Serial Monitor
// ESP32 ADC is 12-bit: values range from 0 to 4095

const int POT_PIN = 34; // GPIO34 is input-only, great for analog

void setup() {
  Serial.begin(115200);  // start serial communication
}

void loop() {
  int rawValue = analogRead(POT_PIN);   // 0 to 4095
  int percent  = map(rawValue, 0, 4095, 0, 100); // convert to 0-100%

  Serial.print("Raw: ");
  Serial.print(rawValue);
  Serial.print("  |  Percent: ");
  Serial.print(percent);
  Serial.println("%");

  delay(300);  // slow down output
}
Open Wokwi's Serial Monitor (bottom panel) and turn the pot. map() is a very useful function — explain how it scales ranges.
04 PWM LED Dimmer (Fade)
output logic
▾

What students learn

PWM (Pulse Width Modulation), duty cycle, how varying the on/off time of a signal controls brightness, and smooth animation using for loops.

Wiring connections

Component PinESP32 PinNotes
LED Anode (+)GPIO 16Through 220Ω resistor
LED Cathode (−)GNDShort leg of LED

Arduino code

Arduino / C++
// Project 4: PWM LED Fade — smooth brightness in and out
// Compatible with ESP32 Arduino Core v3.x (Wokwi default)

const int LED_PIN  = 16;
const int PWM_FREQ = 5000;  // 5 kHz
const int PWM_RES  = 8;     // 8-bit resolution: values 0–255

void setup() {
  ledcAttach(LED_PIN, PWM_FREQ, PWM_RES); // single-call setup (Core v3.x)
}

void loop() {
  // Fade in: dark → bright
  for (int b = 0; b <= 255; b++) {
    ledcWrite(LED_PIN, b);
    delay(8);
  }
  // Fade out: bright → dark
  for (int b = 255; b >= 0; b--) {
    ledcWrite(LED_PIN, b);
    delay(8);
  }
}
Watch the LED smoothly fade in and out. Ask students: if PWM_RES is 8-bit, why do values go from 0 to 255? Because 2⁸ = 256 levels. Try changing delay(8) to delay(2) for a faster pulse effect.
05 Temperature & Humidity (DHT22)
sensor
▾

What students learn

Sensor libraries, single-wire data protocol, reading structured sensor data (temperature + humidity together).

Wiring connections

Component PinESP32 PinNotes
DHT22 VCC3.3VPower
DHT22 DATAGPIO 4Direct connection
DHT22 GNDGND

Library needed

In Wokwi: automatically available. In Arduino IDE: install "DHT sensor library" by Adafruit.

Arduino code

Arduino / C++
// Project 5: DHT22 Temperature & Humidity
// Library: DHT sensor library by Adafruit

#include <DHT.h>

const int DHT_PIN  = 4;
const int DHT_TYPE = DHT22;

DHT dht(DHT_PIN, DHT_TYPE);

void setup() {
  Serial.begin(115200);
  dht.begin();
  Serial.println("DHT22 Ready!");
}

void loop() {
  delay(2000);  // DHT22 needs 2s between readings

  float humidity    = dht.readHumidity();
  float temperature = dht.readTemperature(); // Celsius

  if (isnan(humidity) || isnan(temperature)) {
    Serial.println("Sensor read error!");
    return;
  }

  Serial.print("Temp: ");
  Serial.print(temperature);
  Serial.print(" °C  |  Humidity: ");
  Serial.print(humidity);
  Serial.println("%");
}
In Wokwi, click the DHT22 sensor during simulation to manually change temperature and humidity values — great for live demos!
06 Ultrasonic Distance Meter (HC-SR04)
sensor
▾

What students learn

Trigger/echo pulse timing, pulseIn(), speed of sound, converting microseconds to centimeters.

Wiring connections

Component PinESP32 PinNotes
HC-SR04 VCC5V (VIN)HC-SR04 needs 5V, not 3.3V
HC-SR04 TRIGGPIO 5Trigger pin
HC-SR04 ECHOGPIO 18Echo pin
HC-SR04 GNDGND

Arduino code

Arduino / C++
// Project 6: Ultrasonic Distance Meter
// Formula: distance (cm) = duration × 0.034 / 2

const int TRIG_PIN = 5;
const int ECHO_PIN = 18;

void setup() {
  Serial.begin(115200);
  pinMode(TRIG_PIN, OUTPUT);
  pinMode(ECHO_PIN, INPUT);
}

void loop() {
  // Send 10µs trigger pulse
  digitalWrite(TRIG_PIN, LOW);
  delayMicroseconds(2);
  digitalWrite(TRIG_PIN, HIGH);
  delayMicroseconds(10);
  digitalWrite(TRIG_PIN, LOW);

  // Measure echo duration in microseconds
  long duration = pulseIn(ECHO_PIN, HIGH);

  // Convert to centimeters
  float distance = duration * 0.034 / 2;

  Serial.print("Distance: ");
  Serial.print(distance);
  Serial.println(" cm");

  delay(500);
}
In Wokwi, drag the object in front of the HC-SR04 sensor to change the simulated distance. Ask students to explain why we divide by 2 (sound travels to object AND back).
07 Traffic Light (3 LEDs)

Yellow LED AnodeGPIO 4Through 220Ω resistor

All LED CathodesGNDShare a common GND

Arduino code

Arduino / C++
// Project 7: Traffic Light Sequence

const int RED_PIN    = 2;
const int YELLOW_PIN = 4;
const int GREEN_PIN  = 5;

void allOff() {
  digitalWrite(RED_PIN,    LOW);
  digitalWrite(YELLOW_PIN, LOW);
  digitalWrite(GREEN_PIN,  LOW);
}

void setup() {
  pinMode(RED_PIN,    OUTPUT);
  pinMode(YELLOW_PIN, OUTPUT);
  pinMode(GREEN_PIN,  OUTPUT);
}

void loop() {
  // RED — Stop
  allOff();
  digitalWrite(RED_PIN, HIGH);
  delay(4000);

  // RED + YELLOW — Get ready
  digitalWrite(YELLOW_PIN, HIGH);
  delay(1000);

  // GREEN — Go
  allOff();
  digitalWrite(GREEN_PIN, HIGH);
  delay(4000);

  // YELLOW — Slow down
  allOff();
  digitalWrite(YELLOW_PIN, HIGH);
  delay(1500);
}
Challenge students: add a push button for pedestrian crossing — pressing it should interrupt the cycle and hold RED for 5 seconds.
08 Buzzer Melody Player
output

What students learn

PWM for audio, frequency = musical pitch, storing data in arrays, looping through arrays with for loops.

Wiring connections

Component PinESP32 PinNotes
Buzzer (+)GPIO 18Passive buzzer only (not active)
Buzzer (–)GND

Arduino code

Arduino / C++
// Project 8: Buzzer Melody — Wokwi compatible (Arduino Core v3.x)

const int BUZZER_PIN = 18;

// Note frequencies (Hz)
const int NOTE_C4 = 262;
const int NOTE_D4 = 294;
const int NOTE_E4 = 330;
const int NOTE_F4 = 349;
const int NOTE_G4 = 392;
const int NOTE_A4 = 440;
const int REST    = 0;

// Twinkle Twinkle Little Star
int melody[] = {
  NOTE_C4, NOTE_C4, NOTE_G4, NOTE_G4,
  NOTE_A4, NOTE_A4, NOTE_G4, REST,
  NOTE_F4, NOTE_F4, NOTE_E4, NOTE_E4,
  NOTE_D4, NOTE_D4, NOTE_C4, REST
};

int durations[] = {
  400, 400, 400, 400,
  400, 400, 800, 400,
  400, 400, 400, 400,
  400, 400, 800, 400
};

int noteCount = sizeof(melody) / sizeof(melody[0]);

void setup() {
  pinMode(BUZZER_PIN, OUTPUT);
}

void loop() {
  for (int i = 0; i < noteCount; i++) {
    if (melody[i] == REST) {
      noTone(BUZZER_PIN);
    } else {
      tone(BUZZER_PIN, melody[i], durations[i]);
    }
    delay(durations[i] + 50); // note duration + short gap
    noTone(BUZZER_PIN);
  }
  delay(2000);
}
Explain: higher Hz = higher pitch. 440 Hz is concert A. Students can add their own notes to the arrays to compose a different song!

Arduino code

Arduino / C++
// Project 9: PIR Motion Detector with LED Alert
// PIR output is HIGH when motion detected, LOW otherwise

const int PIR_PIN = 13;
const int LED_PIN = 2;

void setup() {
  Serial.begin(115200);
  pinMode(PIR_PIN, INPUT);
  pinMode(LED_PIN, OUTPUT);
  Serial.println("PIR warming up... wait 30 seconds");
  delay(5000); // In real life PIR needs 30s warm-up; short for demo
  Serial.println("Ready! Watching for motion...");
}

void loop() {
  int motion = digitalRead(PIR_PIN);

  if (motion == HIGH) {
    digitalWrite(LED_PIN, HIGH);
    Serial.println("Motion DETECTED!");
    delay(2000);  // keep LED on for 2 seconds
  } else {
    digitalWrite(LED_PIN, LOW);
    Serial.println("No motion.");
  }

  delay(200);
}
In Wokwi, click the PIR sensor during simulation to toggle motion ON/OFF. Add a buzzer to make it a real intruder alarm!
10 LDR Light Sensor + Auto LED
sensor output logic
▾

What students learn

Analog sensors, voltage divider, threshold-based automation — the pattern behind smart lighting, auto-brightness, and countless IoT devices.

Wiring connections

Component PinESP32 PinNotes
LDR Pin 13.3VEither leg of the LDR
LDR Pin 2GPIO 34Other leg of the LDR
10kΩ Resistor Pin 1GPIO 34Same row as LDR Pin 2
10kΩ Resistor Pin 2GNDPulls the signal down
LED Anode (+)GPIO 2Through 220Ω resistor
LED Cathode (−)GNDShort leg of LED
Think of the LDR and the 10kΩ resistor as a voltage divider — they share a middle point connected to GPIO 34. When it's bright, LDR resistance drops and the voltage at GPIO 34 goes low. When dark, resistance rises and voltage goes high. The ESP32 reads this change.

Arduino code

Arduino / C++
// Project 10: LDR Auto LED — LED turns ON in the dark
// Lower LDR value = more light. Higher value = darker.

const int LDR_PIN   = 34;
const int LED_PIN   = 2;
const int THRESHOLD = 2000; // adjust based on your environment

void setup() {
  Serial.begin(115200);
  pinMode(LED_PIN, OUTPUT);
}

void loop() {
  int ldrValue = analogRead(LDR_PIN);  // 0–4095

  Serial.print("Light level: ");
  Serial.print(ldrValue);

  if (ldrValue > THRESHOLD) {
    // It's dark — LDR resistance high, voltage high
    digitalWrite(LED_PIN, HIGH);
    Serial.println("  → DARK, LED ON");
  } else {
    // It's bright
    digitalWrite(LED_PIN, LOW);
    Serial.println("  → BRIGHT, LED OFF");
  }

  delay(500);
}
Run it first and read the raw LDR value in serial — then set THRESHOLD to a value between your bright and dark readings. In Wokwi, slide the LDR light slider to simulate day/night.
